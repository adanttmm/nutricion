# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project

AI-powered nutritionist assistant. Reads scanned PDF diet plans from `./Dietas/` (via Claude Vision), extracts structured nutrition data, calculates caloric intake, and generates weekly menus, Costco+City Market shopping lists, recipes with video links, and weekend meal-prep schedules.

## Setup

```bash
bash setup.sh            # creates venv, installs deps, copies .env.example → .env
source venv/bin/activate
# Add ANTHROPIC_API_KEY to .env
```

Dependencies include `pymupdf` for rendering scanned PDFs to images before Vision extraction.

## Commands

```bash
# ── TYPICAL WEEKLY WORKFLOW ──────────────────────────────────────────────────

# 1. Drop new PDFs from the nutritionist into Dietas/, then parse them:
python main.py parsear-dietas
#    Reads Dietas/YYYYMMDD_ATM.pdf + Dietas/YYYYMMDD_IOB.pdf via Claude Vision
#    Saves config/parsed_diets/YYYYMMDD_ATM.yaml, YYYYMMDD_IOB.yaml, combined_YYYYMMDD.yaml

# 2. Generate everything for the week (auto-picks the latest parsed plan):
python main.py semana-completa
#    outputs/menus/menu_YYYY-MM-DD.md
#    outputs/shopping/compras_YYYY-MM-DD.md   (Costco first, City Market supplement)
#    outputs/recipes/recetas_YYYY-MM-DD.md    (full recipes + YouTube search links)
#    outputs/meal_prep/meal_prep_YYYY-MM-DD.md

# ── OTHER USEFUL COMMANDS ────────────────────────────────────────────────────

# Parse only one person:
python main.py parsear-dietas --persona ATM

# Explicit plan or date:
python main.py semana-completa --plan config/parsed_diets/combined_20260604.yaml --semana 2026-06-09

# Step by step:
python main.py generar-menu      --plan config/parsed_diets/combined_20260604.yaml
python main.py generar-compras   --menu outputs/menus/menu_2026-06-09.md
python main.py generar-recetas   --menu outputs/menus/menu_2026-06-09.md
python main.py planear-prep      --menu outputs/menus/menu_2026-06-09.md --recetas outputs/recipes/recetas_2026-06-09.md

# Single recipe:
python main.py receta "Salmón en costra de hierbas"

# Daily tracking:
python main.py registrar
python main.py resumen
python main.py peso --kg 80.5
python main.py graficas --semana 2026-06-09
```

## Architecture

```
Dietas/                    # Drop PDFs here: YYYYMMDD_ATM.pdf, YYYYMMDD_IOB.pdf
config/
  user_profile.yaml        # household, activity, shopping preferences
  diet_plan_example.yaml   # fallback template if no PDFs have been parsed yet
  stores.yaml              # Costco / City Market sections
  parsed_diets/            # Auto-generated by parsear-dietas
    YYYYMMDD_ATM.yaml      # parsed plan for person ATM
    YYYYMMDD_IOB.yaml      # parsed plan / body composition for IOB
    combined_YYYYMMDD.yaml # merged household plan used by menu generator

skills/
  base_skill.py            # Anthropic client, prompt caching, file save
  diet_parser.py           # Renders PDF pages → Claude Vision → structured YAML + macro calc
  menu_generator.py        # 7-day varied gourmet menu; understands both parsed and template YAMLs
  shopping_list.py         # Organized by Costco/City Market section, quantities for 2-3 people
  recipe_finder.py         # Full recipes + YouTube/Google search URLs
  meal_prep_planner.py     # Weekend prep schedule, parallel tasks, storage guide

tracker/
  daily_log.py             # SQLite: meals, daily_goals, body_metrics

visualizations/
  charts.py                # Matplotlib: weekly calorie bar, macro donut, weight trend

main.py                    # Click CLI; skills are imported lazily per command
outputs/                   # Generated Markdown + PNG (not committed)
data/                      # SQLite DB + menu_history.txt for variety across weeks
```

## Key design decisions

- **PDF files are scanned images** (CamScanner) — `pymupdf` renders each page to PNG at 2× resolution; `DietParserSkill` sends all pages as base64 images to Claude Vision in a single API call.
- **Document type detection**: the parser distinguishes diet plans (with meals) from body composition reports (measurements only). IOB's current file is a body composition report; a diet plan for IOB can be added later.
- **Combined plan**: when both ATM and IOB have diet plans, `create_combined_plan()` asks Claude to merge them into a single household plan. If only one has a diet plan, that plan is used directly with body comp data attached as reference.
- **Menu generator is format-agnostic**: `_is_parsed_diet()` checks for `foods[]` lists in `meal_structure`; if found, it presents the nutritionist's exact foods as the model to vary — not just calorie targets.
- **Skills use prompt caching** (`cache_control: ephemeral`) on all system prompts.
- **Menu history** (`data/menu_history.txt`) accumulates dish names so the AI avoids repeating meals across weeks.
- **Recipe video links** are dynamic YouTube/Google search URLs, always valid.

## Naming convention for Dietas/ files

```
YYYYMMDD_INICIALES.pdf
  e.g. 20260609_ATM.pdf   ← Adán's diet plan for week of June 9
       20260609_IOB.pdf   ← IOB's plan or body composition for same week
```

The parser always picks the **most recent** file per person (by date in filename).

## User context (encoded in all skill prompts)

- Cooks for **2 people** all meals; **3rd person at lunch Mon/Wed/Fri**
- **1 cheat meal/week** (Saturday dinner by default)
- Gourmet style, all cuisines, advanced technique
- Gym 3×/week (Mon/Wed/Fri), trekking 45 min daily, salsa 2×/week (Tue/Thu)
- Gym days +150 kcal, salsa days +100 kcal (applied to colación PM)
- Primary shopping: **Costco** (Interlomas / Santa Fe); specialty at **City Market Santa Fe**
